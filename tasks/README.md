# Tasks Directory

This directory is the shared communication channel between the independent Review Agent in ChatGPT Tasks and the Codex Builder Agent.

## File roles

- `NEXT_TASK.md`: the only active task file Codex should execute.
- `YYYYMMDD_HHMMSS_T###_<slug>.md`: immutable archived task versions generated by the Review Agent.
- `TASK_INDEX.md`: chronological index of task versions, review sources, current status, and important paths.
- `CODEX_BUILDER_BOOTSTRAP.md`: startup instructions for a new Codex Builder Agent.

## Required workflow

1. Review Agent inspects repository changes and writes a new versioned task file.
2. Review Agent updates `NEXT_TASK.md` to match the newest active versioned task.
3. Codex Builder pulls the repository, reads `tasks/CODEX_BUILDER_BOOTSTRAP.md`, then executes only `tasks/NEXT_TASK.md`.
4. Codex Builder writes all outputs back to the repository with explicit paths, logs, reports, artifact manifests, and status updates.
5. Review Agent reads the new repository state on the next scheduled run and generates the following task.

## Stage policy

The user has authorized the Review Agent to allow Stage 6 without separate human approval if the repository evidence supports it. However, Stage 6 must still be gated by explicit written evidence in `NEXT_TASK.md`; Codex must not infer permission from older README text or chat history.
